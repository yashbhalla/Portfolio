The NX-AddOns a suite of supplementary CAD tools customized to optimize the performance of our wiring harness design team in India. These were crafted to align seamlessly with the specifications provided by our esteemed German RD counterparts. I implemented 3 NX Add-Ons from scratch, using C#, and was responsible for the overall bug fixes, future enhancement and renovation of 12 NX Add-Ons out of 60 tools tailored for the RD colleagues.

I conceived, designed, and developed Tool Package, a WinForms Desktop Application, using C# (.Net Framework & .Net Core), WinForms, HTML 5, CSS 3, and XML. Being active in a CI/CD workflow, we implemented new enhancements regularly. However, the manual process of transferring essential files and folders to the shared drive proved to be error-prone, causing unnecessary bug fix tickets. Thus, the development of Tool Package enhanced the efficiency of the CI/CD process by offering a seamless one-click solution. It reduced the delivery time by an impressive 6000%, nullified redundant tickets, and got nominated for "My First Innovation" under Product and Business Innovation categories.

I pioneered Smoke Test, a WinForms Desktop Application, built on C# (.Net Framework) and WinForms, integrated with UiPath to automate testing of over 60 NX Add-On tools. The application runs all the tools on a master harness, generates a .CSV file with the test results of each tool, and captures screenshots of all the runs, whether the tool ran successfully or encountered errors, in a new "SmokeTestResult" folder. This crucial application is used every six months in the NX roll-out process to identify the faulty tools and promptly inform the developers.
